Week 02 Notes

Program Structure

  1. In some cases, JavaScript allows you to omit the semicolon at the end of a statement. In other cases, it has to be
    there, or the next line will be treated as part of the same statement. The rules for when it can be safely omitted are
    somewhat complex and error-prone. So in this book, every statement that needs a semicolon will always get one.
  2. full list of keywords and reserved words is rather long: break case catch class const continue debugger default delete

    private protected public return static super switch this throw true try typeof var void while with yield
